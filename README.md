## Цель работы:
- научиться выбирать правильный тип функции для задачи, понимать области видимости переменных (Scope), освоить замыкания (Closures), разграничивать доступ к переменным, применять функции для решения практических задач и подготовиться к вопросам нп Junior-собеседовании

### Используемые технологии в работе:
- HTML:
В проекте использована современная разметка для создания интерфейса калькуляторов:
•	Теги секций (<section>, <div>): Использованы для логического разделения заданий на карточки (card), что позволяет изолировать код каждого блока.
•	Элементы форм:
<input type="number"> — для ввода числовых параметров (вес, рост, возраст).
<select> — для выбора пола в расчете BMR.
<button> — для инициализации вызова функций.
•	Атрибут title: Применен для создания всплывающих подсказок (tooltips) с информацией о допустимых диапазонах ввода.

- CSS:
Для стилизации приложения применены следующие технологии:
•	CSS Variables (Переменные): позволили реализовать динамическую смену тем (светлая/темная) через изменение значений цветов в одном месте («:root»).
•	Flexbox Layout: обеспечил вертикальное расположение полей («один под другим») и выравнивание кнопок.
•	Transitions (Переходы): использованы для плавного изменения цвета рамки при валидации и плавного появления плашек с результатом.
•	Визуальная валидация: реализована через динамическую смену классов «.success» (зеленая рамка) и «.error» (красная рамка), что дает мгновенный фидбек пользователю.

- JS:
В проекте продемонстрированы все основные способы создания функций:
•	Function Declaration (Объявление функции):
Пример: function calculateBMI(...)
Технология: использован Hoisting (всплытие), что позволяет вызывать эти функции в коде выше места их фактического написания.
•	Function Expression (Функциональное выражение):
Пример: const toLbs = function(...)
Технология: функция создается как значение переменной. У таких функций нет всплытия, поэтому их можно вызывать только после объявления.
•	Arrow Functions (Стрелочные функции):
Пример: const cmToFt = (cm) => ...
Технология: использован современный короткий синтаксис (ES6). В Задании 4 они применены для лаконичных вычислений без использования ключевого слова return в одну строку.
•	Closures (Замыкания):
Технология: реализована функция-валидатор, которая «запоминает» минимальные и максимальные границы диапазона из своей внешней области видимости.
•	Scope (Область видимости):
Global Scope: глобальные константы (например, коэффициенты) доступны всем функциям через лексическое окружение.
Function Scope: локальные переменные внутри расчетов защищены от изменений извне и не «загрязняют» глобальную область.
Реализованные функции по уровням:
1.	Базовый: расчет ИМТ и конвертация веса с проверкой на положительные числа.
2.	Junior: расчет нормы калорий (BMR) с использованием глобальных констант и сложная конвертация роста в формат 5'9".
3.	PRO: валидация данных через механизм замыканий и демонстрация уровней видимости переменных в консоли.
Для корректной работы «живой» валидации использован глобальный обработчик событий «addEventListener('input', ...)», который мгновенно проверяет данные согласно правилам из лекции: «одна функция – одно действие».

### Созданные в работе функции:
Данный проект представляет собой «Калькулятор здоровья и фитнеса», объединяющий в себе несколько инструментов для расчета параметров тела и конвертации единиц измерения.
В проекте реализованы следующие функции (согласно принципу Single Responsibility):
1. «calculateBMI (weight, height)» (Function Declaration) – рассчитывает индекс массы тела. Выбрана из-за поддержки «hoisting», что позволяет вызывать её в любом месте скрипта.
2. «kgToLbs» / «lbsToKg» (Function Expression) – конвертеры веса. Выбраны для обеспечения строгого порядка кода (недоступны до объявления).
3. «calculateBMR (weight, height, age, gender)» – рассчитывает базальный метаболизм по формуле Миффлина-Сан Жеора. Демонстрирует работу с «Lexical Environment» (использование внешней константы «BMR_COEFFICIENT»).
4. «cmToFtIn» / «ftInToCm» (Arrow Functions) – краткие функции в одну строку для конвертации роста. Выбраны за лаконичный синтаксис.
5. Функции-обработчики («handleBMI», «handleBMR» и др.) – отвечают за сбор данных с экрана, их «валидацию» и вывод результата пользователю.



#### Ответы на контрольные вопросы (вопросы для интервью):
- Что произойдет, если в функции нет оператора return?
  Ключевое слово return отправляет результат работы функции обратно в место вызова. Если в теле функции оператор return отсутствует, функция выполнит весь заложенный в неё код и по умолчанию вернет значение undefined.

- Почему стрелочные функции нельзя вызвать до ее объявления?
  Стрелочная функция (Arrow Function) представляет собой современный синтаксис функционального выражения. Как и в случае с Function Expression, для них нет hoisting (всплытия). Поскольку функция создается как значение переменной (обычно const), она инициализируется только в момент достижения интерпретатором строки кода с её объявлением.

- Что такое «Замыкание» (Closure) простыми словами?
  Замыкание – это ситуация, когда внутренняя функция «запоминает» переменные из своей внешней области видимости. Она продолжает «помнить» и иметь доступ к этим переменным даже после того, как внешняя функция завершила свое выполнение. В нашей работе это использовано в Задании 5 для создания валидатора с фиксированными границами диапазона.

- Зачем нужен принцип «одна функция – одна задача» (Single Responsiblity)?
  Согласно теории и практики, функция должна делать только то, что явно подразумевается её названием. Это помогает лучше структурировать код, избавляться от запутанных частей и делать функции независимыми. Например, функция проверки данных (validate) должна только проверять их, а не выводить сообщения об ошибках.

- Чем отличается Function Declaration от Function Expression?
  Главное отличие в «hoisting» (всплытии). Declaration создается при запуске скрипта и доступна везде. Expression создается только тогда, когда поток выполнения доходит до неё.

- Что такое hoisting для функции?
  Hoisting – это механизм в JavaScript, который «поднимает» объявления функций (Function Declaration) наверх в начало области видимости. Это происходит потому, что JS «читает» весь код перед выполнением и находит все FD, позволяя вызывать их в любом месте программы

- Почему стрелочные функции нельзя вызвать до объявления?
  Они всегда являются анонимными и обычно присваиваются переменным («const» или «let»). Переменные не поддерживают hoisting в той же мере, что и Function Declaration, поэтому вызов до строки инициализации приведет к ошибке.

- Что такое лексическое окружение?
  Это внутренний механизм JS, который хранит данные функции и ссылку на внешнее окружение. Именно благодаря ему функция "знает", какие переменные ей доступны снаружи.

- Почему важно комментировать выбор типа функции?
  Выбор типа функции (FD, FE или AF) в JavaScript напрямую влияет на архитектуру: использование Function Declaration позволяет гибко располагать код, а Arrow Functions обеспечивают лаконичность для простых операций. Комментирование выбора показывает понимание таких нюансов, как hoisting и контекст выполнения, что критически важно для командной разработки и поддержки кода.

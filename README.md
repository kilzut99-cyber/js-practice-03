#### Ответы на контрольные вопросы:

- В чем разница между ==и===? Приведите пример, где == может дать неожиданный результат.
  Логический оператор «==» (нестрогое равенство) сравнивает значения его операндов с приведением (преобразованием) их типов, а «===» (строгое соответствие, оператор идентичности) – сравнивает значения и типы операндов без приведения, руководствуясь строгим определением совпадения.
  Оператор «==» часто приводит к неожиданным багам, делая неявные преобразования перед сравнением. например, согласно «правил преобразования типов», 0 == false // true; "" == false // true; null == undefined // true; " 1 " == true // true.

- Что такое приведение типов (type coercion)? Что получится в выражениях: 5+"5" и 5-"5"?
  Приведение типов (type coercion) в JavaScript – это автоматическое или неявное преобразование значений из одного типа данных в другой (например, строки в число) при выполнении операций. Такое преобразование обусловлено слабой типизацией JS: переменные могут содержать значения разных типов данных (например, число, строку или объект) без явного указания типа:
  alert (5+”5”); // “55”
  alert (5-”5”); // 0

- Когда уместно использовать switch, а когда лучше if/else?
  Инструкция switch используется для выбора одного из нескольких вариантов действий в зависимости от результата сравнения значения выражения с несколькими константами. Инструкцию if уместно использовать когда нет необходимости использования вложенных инструкций
  if/else.

- Что вернет prompt(), если пользовать нажмет "Отмена"? Как вы это обрабатываете в своем квизе?
  Функция prompt() возвращает, то что ввел опрашиваемый в поле (строку) для ввода текста (например, количество лет). При нажатии «Отмена» выводится специальное значение null (Safari 5.1+ не возвращает null). В моем «Квизе», если пользователь отменил prompt – квиз завершается и выводится «Квиз отменен».

- Как проверить, что в переменной действително число, а не NaN?
  Самый надежный способ в JavaScript проверить, что переменная является числом и не равна NaN, – использовать метод Number.isNaN(variable) для выявления NaN, в сочетании с проверкой типа данных typeof variable === 'number'. Также полезно использовать Number.isFinite() для исключения Infinity.

- Что такое Early Return и чем он лучше вложенных if/else?
  Early Return (ранний возврат) – это техника программирования, при которой функция завершается досрочно, как только выполняется определенное (обычно ошибочное) условие, что позволяет избежать вложенных конструкций if-else. Вместо глубокой вложенности используются «защитные условия» (guard clauses). Это делает код чище, проще и читабельнее.
  Когда использовать:
  • Когда необходимо проверить входящие параметры на валидность.
  • Когда функция имеет несколько условий, блокирующих ее выполнение.
  • Для упрощения сложных логических ветвлений.

- Зачем нужен блок default в операторе switch?
  Блок default выполняется в том случае, если не найдено ни одного соответствия с метками блоков case. Наличие блока default не обязательно, если его нет и не найдено ни одного соответствия, то никакие инструкции выполнены не будут.
